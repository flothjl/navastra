# Product Brief

## One-Liner

Navastra is a personal web namespace that lets people jump to important places
with short, memorable names.

## Problem

People lose time navigating to the same digital places over and over:

- important docs buried in chat threads
- admin panels behind multiple dashboards
- local tools with awkward URLs
- shared references that are hard to rediscover
- browser bookmarks that become a drawer full of junk

The real problem is not storage. It is recall.

People remember names better than URLs. They want to type the name they know and
arrive at the thing they mean.

## Target Users

### Primary

Individuals who live in browsers all day:

- developers
- founders
- operators
- analysts
- power users
- remote workers

### Secondary

Small groups that need lightweight shared navigation:

- families
- friend groups
- open-source projects
- small teams
- clubs and communities

## Core Promise

If you can remember the name, Navastra can get you there.

## Jobs To Be Done

- When I need a frequently used destination, I want to type a short name so I
  can get there without searching.
- When I set up a new device, I want my navigation namespace to follow me.
- When I work with another person, I want to share a small set of useful routes
  without exposing all of my personal links.
- When a link changes, I want to update it once and keep the old mental handle.
- When I leave a platform or browser, I want my route names and destinations to
  remain mine.

## Differentiation

Enterprise go-link products are usually workspace tools. Navastra starts with
the individual and uses Enbox to make that individual ownership real.

The product should feel like a faster bookmark manager, but the deeper
differentiator is ownership:

- the namespace belongs to the user
- routes are portable Enbox records
- sync is backed by user-controlled identity and DWN storage
- sharing can be explicit and scoped

The product should not lead with "self-sovereign" language. It should lead with
speed, memory, and control. Enbox is the proof mechanism, not the billboard.

## Positioning

Navastra is not:

- a full browser
- a generic bookmark manager
- a search engine
- a team intranet
- a read-it-later app

Navastra is:

- a personal command layer for web navigation
- a portable namespace for links
- a fast recall tool
- a small shared map when people choose to collaborate
- a reference app for Enbox's protocol-first application model

## Success Metrics

Early product validation should focus on behavior:

- number of routes created per active user
- daily route resolutions
- percentage of routes resolved from address bar or extension
- time from install to first successful jump
- weekly retained users
- number of imported bookmarks converted into active routes
- number of shared route collections created
